# 注意事项

# 环境要求

  1. 当前只支持Linux，不支持Windows和虚拟空间

  2. PHP版本：需要大于等于7.1版本

  3. 安装 Redis：插件防盗刷CDN流量的功能会使用到Redis

  4. 插件有加密压缩文件的功能，phpzip扩展不支持加密压缩、压缩性能也不好，需要用到服务器系统的zip工具

  5. 启用PHP的exec函数：因为上一条的原因，需要使用到PHP的exec函数去执行zip压缩和加密

警告

如果是用b2主题，那么必须开启主题的cookie兼容，不然用户没法下载

# 安装和配置

# 安装

在 WordPress 后台上传安装、启用插件，安装的时候如有提示，请参考提示进行环境配置

# 又拍云配置

  1. 登录又拍云管理控制台，创建云存储服务和授权操作员：

    操作员的权限必须能读、写、删

  2. 配置时间戳防盗链

    打开访问控制，找到Token 防盗链，开启服务

  3. 配置跨域（前端直传需要用到） 打开访问控制，找到 CORS 跨域共享，开启服务 Allow Headers： Access-Control-Allow-Origin

  4. 配置 cdn 加速域名和解析域名

    先到又拍云的”域名管理“里面添加 cdn 加速域名

    然后复制 CNAME 解析值到你的域名解析服务商那边添加一条 CNAME 解析

    又拍云云存储和CDN加速域名配置好了之后，就可以在WordPress后台的插件设置页面进行配置了

# 七牛云配置

  1. 登录七牛云管理控制台，打开对象存储，创建存储空间：

    访问控制：私有

  2. 配置CDN加速域名和解析域名 先到七牛云的CDN域名管理里面添加CDN加速域名

    使用场景：下载分发

    源站配置：七牛云存储，选择你刚才创建的私有云存储空间

    其他配置项默认就行了

    按照提示配置 CNAME 解析，参考上面又拍云的配置

    然后配置Referer防盗链 打开CDN域名管理，选择你刚才绑定的域名，点开之后往下拉，找到“访问控制”，修改Referer防盗链的配置

    配置跨域，选择你刚才创建的空间，打开空间配置，找到“跨域配置”，添加规则，把自己的域名加上去

# 阿里云配置

  1. 登录阿里云管理控制台，创建 Bucket

    读写权限：私有

    其他的配置，你们可以根据自己具体的需求进行选择

  2. 接下来配置账号的访问控制权限

    选择刚才创建的bucket空间，然后点击左边菜单栏的权限管理，进入如下页面

    创建一个专门用来管理OSS空间的用户

    访问方式：编程访问

    创建用户之后会显示这个用户对应的密钥对，自己得保存一下密钥对，以后不会再显示密钥对了

    然后回到上一步的页面，进行授权操作

  3. 在传输管理那边绑定 cdn加速域名，按照提示进行 CNAME 解析，参考上面又拍云的配置

  4. 配置跨域 选择刚才创建的bucket空间，然后点击左边菜单栏的权限管理，选择 “跨域设置” 进入如下页面 添加规则

# 腾讯云配置

  1. 登录腾讯云对象存储管理控制台，创建 Bucket

    读写权限：私有

    其他的配置，你们可以根据自己具体的需求进行选择

  2. 接下来配置账号的访问控制权限

    选择刚才创建的bucket空间，然后点击左边菜单栏的权限管理，进入如下页面，先给腾讯云CDN服务加一个权限

    到腾讯云访问管理快速创建一个专门用来管理COS空间的用户

    访问方式：编程访问

    策略：QcloudCOSFullAccess

    创建用户之后回到用户列表，选择刚才创建的用户，配置API 密钥

  3. 在传输管理那边绑定 cdn加速域名，按照提示进行 CNAME 解析，参考上面又拍云的配置

    鉴权配置： 在域名管理选择对应的cdn加速域名，然后切换到“访问控制”tab页找到鉴权配置

    注意

    鉴权模式 选择 TypeD

  4. 配置跨域 选择刚才创建的存储桶，然后点击左边菜单栏的安全管理，选择 “跨域访问CORS设置”
